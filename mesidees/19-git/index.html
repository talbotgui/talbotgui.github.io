Git

Les basics :

Pour cloner un dépot : ssh git clone https://xxxxxxxxx

Pour ajouter une modification : ssh git add rep1/monFichier.xxx

Pour ajouter toutes les modifications : ssh git add -A

Pour annuler les ADD : ssh git reset

Pour annuler le ADD d’un fichier : ssh git reset rep1/monFichier.xxx

Pour annuler les modifications d’un fichier : ssh git checkout rep1/monFichier.xxx

Pour commiter : ssh git commit -m "mon message"

Pour commiter les fichiers modifiés (pas les ajouts) sans faire de ADD : ssh git commit -am "mon message"

Pour pousser ces commits sur le repository : ssh git push

Pour récupérer toutes les modifications réalisées depuis le dernier PULL : ssh git pull ou ssh git pull --rebase si des commits ont été réalisés depuis le dernier PULL

En cas de problème durant un rebase, la commande ssh git rebase --abort annule tout.

Les utilitaires :

Pour voir le statut du dépot local : ssh git status

Pour voir les modifications d’un fichier : ssh git diff rep1/monFichier.xxx

Pour voir toutes les modifications : ssh git diff

Pour obtenir les logs : ssh git log ou ssh git log --pretty=oneline

Pour afficher les informations du repository distant : ssh git remote show origin

Pour sauvegarder dans un espace temporaire des modifications (sans les commiter) : ssh git stash

Pour lister les sauvegardes : ssh git stash list

Pour récupérer la sauvegarde la plus récente : ssh git stash pop

Pour annuler tous les commits locaux : ssh git reset HEAD^

La manipulation d’historique :

Pour modifier l’historique sur les 5 derniers commits (à ne faire que si aucun push n’a été fait depuis) : ssh git rebase --interactive HEAD~5

Dans l’éditeur qui s’ouvre, remplacer les pick par :

  • r pour reword afin de modifier le message du commit (le nouveau message est à saisir après avoir sauvegarder ce premier éditeur de texte)
  • s pour squash permet de fusionner le commit avec celui qui le précède dans le temps (et fusionne les commentaires des deux commits)
  • f pour fixup qui fusionne comme le squash mais qui supprime le commentaire

Pour pousser sur le repository distant un rebase qui modifie l’historique déjà poussé /!\ attention quand on travaille en équipe /!\ : ssh git push --force

La manipulation de branches :

Pour lister les branches locales : ssh git branch

Pour passer d’une branche à l’autre : ssh git checkout maBranche

Pour créer et passer à une autre branche : ssh git checkout -b maBranche

Pour supprimer une branche : ssh git branch -d maBranche

Pour afficher les différences entre deux branches : ssh git dif maBranche...lAutreBranche

Pour fusionner simplement une branche vers la branche courante : ssh git merge monAutreBranche

Pour pousser dans le master les modifications d’une autre branche en un seul commit : ssh git checkout master git merge --no-ff maBrancheFeature

Pour pousser sur le dépot distant toutes les branches : ssh git push --all

Pour lier une branche locale à une branche distante existante (après avoir fait un push -all créant une branche par exemple) : ssh git branch --set-upstream-to origin/maBranche
